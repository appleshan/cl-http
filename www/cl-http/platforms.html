Allegro Common Lisp

  • ACL (JKF) 5.0, 6.0, 7.0, 8.0: CL-HTTP runs under Allegro Common Lisp (ACL) for UNIX, Mac OS X and Windows. The server is considered beta-test software running at the HTTP 1.1 level. Contact Franz, Inc. for information on how to obtain their lisp.

  • ACL (OBC) 4.2, 4.3, 5.0, 6.0, 7.0, 8.0: At present, CL-HTTP runs in multi-threaded mode under Allegro Common Lisp (ACL) for UNIX, and has been tested under SunOS 4.1.3x, Solaris 2.4 and 2.5 on Sun Microsystems workstations. The ACL CL-HTTP server is considered released software when running at the HTTP 1.0 level under ACL 4.2 on SunOS 4.1.3x. CL-HTTP also runs as beta test software under Allegro Common Lisp for Windows. Contact Franz, Inc. for information on how to obtain their lisp.

  • Bug Reports: If you encounter bugs, unclear documentation, or other shortcomings, please report them to bug-acl-cl-http@nospam.cl-http.org with a description of how to reproduce the problem and the versions of CL-HTTP, ACL, and the operating system you were using.

CMU Common Lisp

  • CMU CL 18b: CL-HTTP requires the 18b release of CMU Common Lisp, and is considered beta test software when running in multi-threaded mode on the x86 port and alpha test software when running in single-threaded mode on the other ports. Recent binary distributions for CMU Common Lisp are available from the CMU CL Source and binaries home site, or from its mirrored site.. CMU CL mailing lists are available for help with CMU CL or discussion of implementation issues.

  • Bug Reports: If you encounter bugs, unclear documentation, or other shortcomings, please report them to bug-cmucl-cl-http@nospam.cl-http.org with a description of how to reproduce the problem and the versions of CL-HTTP, CMU CL, and the operating system you were using.

LispWorks Common Lisp

  • LispWorks 5.0, 5.1 for UNIX, Microsoft Windows, UNIX and Macintosh: CL-HTTP is considered released software when running on unix, Windows Vista/XP/2000/NT, Windows 95/98/Me, Linux and Mac OS X 10.5. Contact LispWorks Ltd. for information on how to obtain their lisp.

  • LispWorks 4.3, 4.4 for UNIX, Microsoft Windows, UNIX and Macintosh: CL-HTTP is considered released software when running multi-threaded under LWW 4.3, 4.4 on unix, Windows NT 4.0/2000/XP, Windows 95/98/Me, Linux and Mac OS X 10.4. Contact Xanalys Inc. for information on how to obtain their lisp.

  • LispWorks 4.2 for UNIX, Microsoft Windows and Linux: CL-HTTP is considered released software when running multi-threaded under LWW 4.2 on unix, Windows NT 4.0/2000/XP, Windows 95/98/Me and Linux. Contact LispWorks Ltd. for information on how to obtain their lisp.

  • LispWorks 4.1 for UNIX, Microsoft Windows and Linux: CL-HTTP is considered released software when running multi-threaded under LWW 4.1 on unix, Windows NT 4.0/2000/XP, Windows 95/98/Me and Linux. Contact LispWorks Ltd. for information on how to obtain their lisp.

  • LispWorks 4.0.1 for Microsoft Windows: CL-HTTP is considered released software when running multi-threaded under LWW 4.0.1 on Windows NT 4.0 and Windows 95/98.
  • Bug Reports: If you encounter bugs, unclear documentation, or other shortcomings, please report them to bug-lw-cl-http@nospam.cl-http.org with a description of how to reproduce the problem and the versions of CL-HTTP, LW, and the operating system you were using.

Lucid Common Lisp

  • LCL 4.2: At present, CL-HTTP runs in multi-threaded mode under Lucid Common Lisp (LCL) for UNIX. The LCL CL-HTTP server is considered beta test software when running under LCL 4.2 on Sparc/Solaris 5.4. This port has not been maintained in recent years. Contact LispWorks Ltd. for information on how to obtain their lisp.

  • Bug Reports: If you encounter bugs, unclear documentation, or other shortcomings, please report them to bug-lcl-cl-http@nospam.cl-http.org with a description of how to reproduce the problem and the versions of CL-HTTP, ACL, and the operating system you were using.

Macintosh Common Lisp

  • MCL 5.0, 5.1: CL-HTTP runs at near production quality under Mac OS X 10.4 and at advanced Beta under Mac OS 9.

  • MCL 4.x (3.x 68k): CL-HTTP runs in multi-threaded mode under MCL 4.0, 4.1, 4.2, 4.3 the PowerPC native version of MCL, its companion 3.1, 3.2, 3.3, 3.4 for 68K machines.The MCL CL-HTTP server is considered released software when running under current versions of Macintosh Common Lisp. Contact Digitool, Inc. for information on how to upgrade to MCL 4.3. The comparable version for 68K Macs is MCL 3.4.

  • MCL 3.9: CL-HTTP runs in multi-threaded mode under MCL 3.9, the PowerPC native version of MCL.

  • MCL 3.0: CL-HTTP also runs in multi-threaded mode under MCL 3.0. The MCL CL-HTTP server is considered beta test software when running Macintosh Common Lisp 3.0.

    MCL 2.0.1 Although the server does run in single-threaded mode under MCL 2.0.1, MCL CL-HTTP is unsupported and considered alpha test in versions prior to MCL 3.0.

  • Bug Reports: If you encounter bugs, unclear documentation, or other shortcomings, please report them to bug-mcl-cl-http@nospam.cl-http.org with a description of how to reproduce the problem and the version of CL-HTTP you were using. Ideally, use the distributed sources to fix the problem and send us the fix along with the report.

The Scieneer Common Lisp

  • Scieneer CL 1.2.11: CL-HTTP runs in multi-threaded mode on the Scieneer Common Lisp implementation and supports multi-processor UNIX systems, and is considered beta test software. The 1.2.11 maintenance release of the Scieneer Common Lisp is required. Contact Scieneer Pty Ltd for information on how to obtain their lisp.

  • Bug Reports: If you encounter bugs, unclear documentation, or other shortcomings, please report them to bug-scl-cl-http@nospam.cl-http.org with a description of how to reproduce the problem and the versions of CL-HTTP, SCL, and the operating system you were using.

Symbolics Lisp Machines

  • Genera 8.3: CL-HTTP was originally developed on Symbolics Lisp Machines and the most sophisticated versions continue to run on Ivory and 36xx machines under Genera 8.3 and 8.5 beta.
